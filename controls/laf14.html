Evaluación de Controles y Riesgos
Categorías

LAF14

Descripción del control

El SGBD tiene limitaciones de acceso y no se puede realizar borrados o actualizaciones masivas.

Posible cumplimiento

Explicar y documentar los controles de seguridad que impide esta situación.

Evidencia de cumplimiento
1. Restricción de acceso por roles y privilegios:
  • El acceso directo al Sistema de Gestión de Bases de Datos (SGBD) está limitado exclusivamente al equipo técnico autorizado.
  • Los usuarios finales no tienen acceso directo al SGBD; todas las interacciones se realizan a través de la capa aplicación web, que controla y valida las acciones permitidas según los roles y permisos asignados.
  • Los privilegios administrativos están segmentados y se auditan regularmente para garantizar que no existan accesos innecesarios o permisos mal configurados.
2. Validación y limitación de operaciones masivas:
  • El diseño del SGBD incluye restricciones que previenen la ejecución de consultas que puedan realizar borrados o actualizaciones masivas sin una validación previa.
  • Este tipo de operaciones (si son necesarias para tareas de mantenimiento o soporte) están protegidas mediante procedimientos almacenados que requieren parámetros específicos verificaciones adicionales.
  • En la interfaz de administración de la base de datos, se han implementado capas de seguridad que solicitan autorizaciones explícitas antes de realizar cualquier operación de alto impacto.
3. Registro de auditoría
  • Todas las acciones que impliquen modificaciones en la base de datos, incluso a nivel de la aplicación, se registran en un sistema de auditoría. Esto incluye el detalle de qué usuario realizó la acción, cuándo se realizó y el impacto generado.
  • Este registro asegura la trazabilidad completa de las operaciones y facilita identificar cualquier actividad no autorizada.
4. Prevención desde la interfaz del sistema:
  • Las funcionalidades del sistema están diseñadas para operar sobre registros individuales o lotes controlados, evitando que los usuarios puedan ejecutar manipulaciones masivas que excedan los límites establecidos.
  • Las operaciones sensibles, como eliminar múltiples registros, requieren confirmaciones explícitas y permisos especiales configurados previamente.
