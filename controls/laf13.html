Evaluación de Controles y Riesgos
Categorías

LAF13

Descripción del control

Si el sistema se instala on premise el usuario no tiene acceso a la base de datos.

Posible cumplimiento

Explicar y documentar los controles de seguridad que impide esta situación.

Evidencia de cumplimiento
1. Arquitectura de software y separación de responsabilidades:
  • El sistema está diseñado como una aplicación web que interactúa con la base de datos únicamente a través de un servidor intermedio.
  • El usuario accede al sistema mediante una interfaz web utilizando credenciales específicas, lo que limita su interacción exclusivamente a las funcionalidades proporcionadas por la aplicación.
2. Aislamiento de la base de datos:
  • La base de datos está instalada en el servidor del cliente (on-premise) y no es accesible directamente desde el dispositivo del usuario final.
  • El acceso a la base de datos está restringido a través de configuraciones de red y firewalls, evitando conexiones externas no autorizadas.
3. Sin acceso a archivos del software:
  • El usuario no tiene acceso físico ni lógico a los archivos del software, ya que estos están alojados y ejecutados directamente en el servidor.
  • Esto garantiza que no puedan realizar consultas directas ni explorar configuraciones internas que pudieran comprometer la seguridad de la base de datos.
4. Controles de autenticación y autorización:
  • El acceso al sistema web requiere autenticación mediante credenciales únicas que permiten identificar al usuario.
  • Las acciones permitidas están restringidas a su rol y permisos definidos, evitando cualquier interacción que no esté controlada por la lógica de la aplicación.
