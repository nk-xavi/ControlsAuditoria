Categorías

LAF23

Descripción del control

El sistema no tiene herramientas de modificación del código fuente o bien se controla debidamente que no se pueda afectar a la legislación aplicable.

Posible cumplimiento

Incluir explicación donde el partner asegure esta situación.

Evidencia de cumplimiento
1. Acceso restringido al código fuente:
  • El código fuente está alojado en un sistema de control de versiones seguro con acceso restringido exclusivamente a los desarrolladores autorizados.
  • Los usuarios finales no tienen acceso al código fuente ni a herramientas que puedan modificarlo, ya que el sistema está desplegado como una solución compilada y empaquetada.
2. Prohibición de herramientas de edición directa:
  • El sistema implementado no incluye funcionalidades ni herramientas que permitan a los usuarios finales editar o modificar el código fuente desde la interfaz o el entorno de producción.
  • Esto garantiza que los usuarios no puedan realizar cambios que puedan impactar en la funcionalidad o incumplir la legislación.
3. Cumplimiento de legislación aplicable:
  • El desarrollo del software sigue estándares de seguridad y buenas prácticas de desarrollo para garantizar su conformidad con las leyes y regulaciones aplicables.
4. Protección del entorno de producción:
  • El entorno de producción está protegido contra cambios no autorizados mediante configuraciones de acceso limitado.
  • Solo el equipo técnico designado puede realizar despliegues o actualizaciones, y estas acciones están controladas y documentadas.
