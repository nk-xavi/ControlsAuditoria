Evaluación de Controles y Riesgos
Categorías

LAF23

Descripción del control

El sistema no tiene herramientas de modificación del código fuente o bien se controla debidamente que no se pueda afectar a la legislación aplicable.


Evidencia de cumplimiento
1. Acceso restringido al código fuente:
  • El código fuente del sistema está gestionado a través de un sistema de control de versiones (como Git) alojado en un entorno seguro, con acceso restringido únicamente a los desarrolladores autorizados.
  • Los usuarios finales no tienen acceso nivisibilidad al código fuente, ya que el sistema está desplegado como una aplicación SaaS completamente compilada y empaquetada, lo que evita manipulaciones externas.
2. Prohibición de herramientas de edición directa:
  • La arquitectura del software no incluye funcionalidades que permitan la modificación del código fuente desde la interfaz de usuario o el entorno de producción.
  • Además, las configuraciones de usuario final están diseñadas para limitarse a las opciones de personalización predefinidas, asegurando que no se puedan alterar funcionalidades críticas o vulnerar el cumplimiento normativo.
3. Cumplimiento de legislación aplicable:
  • Durante el desarrollo y las actualizaciones del software, se realizan auditorías internas para verificar que las modificaciones respetan las leyes aplicables, como la Ley Antifraude (Ley 11/2021) y otros reglamentos específicos.
  • Los desarrollos pasan por pruebas en entornos de simulación para evaluar su impacto en el cumplimiento normativo antes de ser implementados en producción.
4. Protección del entorno de producción:
  • El entorno de producción está configurado con accesos altamente restringidos, permitiendo únicamente al equipo técnico autorizado realizar actualizaciones o cambios en el sistema.
5. Mecanismos de verificación de integridad:
  • Se implementan controles automáticos para verificar la integridad del software en el entorno de producción, asegurando que no existan modificaciones no autorizadas.
  • En caso de detectar discrepancias con el código fuente original, se activa un protocolo de revisión y restauración.
